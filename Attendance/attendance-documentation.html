مستندات API سامانه غیبت

این مستندات توضیحات کاملی از تمامی API های سامانه غیبت را فراهم می‌کند که از Django Rest Framework (DRF) و دیگر ابزارهای Django برای مدیریت و ثبت غیبت‌ها استفاده می‌کند.

1. لیست غیبت‌ها

POST /attendance/

این endpoint برای دریافت لیست غیبت‌ها با فیلترهای مختلف استفاده می‌شود.

{
    "student__first_name": "Ali", 
    "student__last_name": "Reza", 
    "student__grade": "11/ریاضی", 
    "student__national_code": "1234567890", 
    "status": "غایب"
}

پاسخ نمونه:

HTTP 200 OK

[
    {
        "student": "متین کچله",
        "date": "2025-02-25",
        "status": "غایب"
    }
]

2. ثبت غیبت جدید

PUT /attendance/create/

این endpoint برای ثبت یا ویرایش غیبت‌های گروهی دانش‌آموزان است. اطلاعات باید به صورت زیر ارسال شوند:

{
    "absences": [
        {
            "national_code": "1234567890",
            "date": "2025-02-25",
            "status": "تاخیر"
        },
        {
            "national_code": "9876543210",
            "date": "2025-02-25",
            "status": "غایب"
        }
    ]
}

پاسخ نمونه:

HTTP 200 OK

{
    "message": "غیبت جدید ثبت شد.",
    "data": [
    {
      "message": "غیبت جدید ثبت شد.",
      "data": {
        "student": "متین کچله",
        "date": "2025-02-26",
        "status": "تاخیر"
      }
    },
    {
      "message": "غیبت قبلی ویرایش شد.",
      "data": {
        "student": "علی کشاورز",
        "date": "2025-02-26",
        "status": "غایب"
      }
    }
  ]
}

3. دریافت غیبت‌های دانش‌آموز بر اساس کد ملی

GET /attendance/detail/str:national_code/

این endpoint برای دریافت تمامی غیبت‌ها برای یک دانش‌آموز بر اساس کد ملی استفاده می‌شود. اگر دانش‌آموز با کد ملی وارد شده پیدا نشود، خطای 404 باز می‌گردد.

GET /attendance/detail/1234567890/

پاسخ نمونه:

HTTP 200 OK

[
    {
        "student": "متین کچله",
        "date": "2025-02-25",
        "status": "غایب"
    }
]

4. گزارش PDF غیبت‌ها

POST /attendance/absences/pdf/

این endpoint برای دریافت گزارش PDF از غیبت‌ها با فیلترهای مختلف استفاده می‌شود.

{
    "student__grade": "11/ریاضی",
    "date_range": {"gte": "2025-02-01", "lte": "2025-02-28"}
}

پاسخ نمونه:

HTTP 200 OK

{
    "file_url": "/media/absence_report.pdf"
}
لای پرانتز: ممکنه این تیکه رو بهتون خروجی نده ولی فایل تو ادرس http://127.0.0.1:8001/media/absence_report.pdf هست

5. گزارش HTML غیبت‌ها

POST /attendance/absences/html/

این endpoint برای دریافت گزارش HTML از غیبت‌ها با فیلترهای مختلف استفاده می‌شود.

{
    "student__grade": "10",
    "date_range": {"gte": "2025-02-01", "lte": "2025-02-28"}
}

پاسخ نمونه:

HTTP 200 OK

{
    "file_url": "/media/absence_report.html"
}
لای پرانتز: ممکنه این تیکه رو بهتون خروجی نده ولی فایل تو ادرس http://127.0.0.1:8001/media/absence_report.html هست

4. جستجوی دانش‌آموزان

POST /attendance/students/

این endpoint برای جستجوی دانش‌آموزان بر اساس فیلترهای مختلف (نام، نام خانوادگی، کد ملی، و پایه تحصیلی) استفاده می‌شود.

برای جستجو، می‌توانید از فیلترهای زیر استفاده کنید:

  • first_name: جستجوی دانش‌آموزان بر اساس نام کوچک
  • last_name: جستجوی دانش‌آموزان بر اساس نام خانوادگی
  • national_code: جستجوی دانش‌آموزان بر اساس کد ملی
  • grade: جستجوی دانش‌آموزان بر اساس پایه تحصیلی

درخواست نمونه:

    {
        "grade": "11"
    }

پاسخ نمونه:

HTTP 200 OK

    [
        {
            "first_name": "Ali",
            "last_name": "Reza",
            "national_code": "1234567890",
            "grade": "11"
        },
        {
            "first_name": "متین",
            "last_name": "دیلاق",
            "national_code": "1234567890",
            "grade": "11"
        }
    ]
